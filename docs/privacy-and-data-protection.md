# Privacy and Data Protection

## Data Collection Summary

| Data Field | Why It Is Needed | Personal Data? | Sensitive Data? | Keep / Remove / Replace | Notes |
|---|---|---|---|---|---|
| Username | Identify the user's account | Yes | No | Keep | Used for login and identifying the account. |
| Email Address | Login and password recovery | Yes | No | Keep | Only used for authentication. |
| Income Amount | Record user income | No | No | Keep | Used for financial tracking. |
| Expense Amount | Record user expenses | No | No | Keep | Used for expense tracking. |
| Expense Category | Organize transactions | No | No | Keep | Helps generate reports and summaries. |
| Transaction Date | Record when transactions occur | No | No | Keep | Used for history and reports. |
| Transaction Note | Optional description of transactions | No | No | Keep | Users may leave this blank. |
| Password | Secure user account | Yes | Yes | Keep (Encrypted) | Passwords must be encrypted and never stored as plain text. |

## Privacy Rule for Prototype

The Budget Tracker Website collects only the information required to manage personal finances, including username, email address, income, expenses, transaction categories, dates, and optional notes. Only registered users can view and manage their own financial records. Passwords are stored securely using encryption. The prototype does not collect unnecessary personal or sensitive information such as national ID numbers, bank account numbers, credit card details, home addresses, or biometric data. The collected data is used only to provide budgeting features and improve the user experience.

## Data Minimization Decision

The following data fields are excluded after the privacy review because they are not necessary for the prototype:

- National ID number
- Bank account number
- Credit card information
- Home address
- Phone number
- Date of birth
- GPS location
- Biometric information (fingerprint or face recognition)
