# Wireframe Specification

## Required Wireframe Screens
Each group must create the following minimum wireframes and export each screen as PNG.

| Wireframe File | Purpose | Minimum Elements to Show | Related Requirements |
|---|---|---|---|
| /wireframes/homepage.png | Entry point of the system | project title, target user, problem, main action buttons, navigation | FR-01, FR-02 |
| /wireframes/input-form.png | User data submission | form fields, required labels, submit button, error/confirmation area | FR-03, FR-10, FR-11 |
| /wireframes/record-list.png | List existing records | table/card list, status, category, search/filter controls | FR-05, FR-06, FR-08 |
| /wireframes/detail-view.png | Show selected record | full details, status, next action | FR-07, FR-08 |
| /wireframes/dashboard.png | Summary/analytics view | totals, categories, status counts, simple chart/card summary | FR-12 |
| /wireframes/admin-view.png | Admin/manager function | record management, update status, approve/edit/delete area | FR-09, FR-08 |

## Wireframe Quality Rules
- Use consistent navigation and labels across screens.
- Avoid decorative-only screens with no functional meaning.
- Show realistic sample data, not empty boxes only.
- Every screen must connect to the final prototype requirements.
- Wireframes may be made using Figma, Canva, PowerPoint, HTML/CSS, or another suitable tool.
