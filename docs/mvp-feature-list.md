# Lab 04 - MVP Feature List

## MVP Decision Rule
The MVP includes only the essential features needed to help users record, manage, and review their daily expenses. Advanced features will be considered in future versions after validating the core problem.

## MoSCoW Prioritization

| Feature ID | Feature Name | Problem Solved | Related User Story | Priority (Must/Should/Could/Won't) | Technical Complexity (1-5) | User Value (1-5) | Evidence Strength (1-5) | Include in Final Prototype? |
|---|---|---|---|---|---:|---:|---:|---|
| F01 | Homepage / Landing Page | Users understand the purpose of the application | US-01 | Must | 1 | 4 | 5 | Yes |
| F02 | Add Expense | Users can record daily expenses | US-02 | Must | 2 | 5 | 5 | Yes |
| F03 | View Expense List | Users can review all recorded expenses | US-03 | Must | 2 | 5 | 5 | Yes |
| F04 | Search / Filter Expenses | Users can quickly find specific expenses | US-04 | Should | 3 | 4 | 4 | Yes |
| F05 | Edit / Delete Expense | Users can correct incorrect records | US-05 | Should | 3 | 4 | 4 | Yes |
| F06 | Confirmation Message | Users know when an action is successful | US-06 | Must | 1 | 3 | 5 | Yes |
| F07 | Dashboard Summary | Users can monitor spending and remaining budget | US-07 | Must | 3 | 5 | 5 | Yes |
| F08 | Admin Record Management | Admin can manage submitted records | US-08 | Could | 4 | 3 | 3 | No |
| F09 | User Login | Protect user accounts | Future | Won't | 4 | 3 | 2 | No |
| F10 | AI Spending Recommendation | Provide smart financial suggestions | Future | Won't | 5 | 3 | 1 | No |

## Must-Have Features
- Homepage / Landing Page
- Add Expense
- View Expense List
- Confirmation Message
- Dashboard Summary

## Should-Have Features
- Search / Filter Expenses
- Edit / Delete Expense

## Could-Have / Future Features
- Admin Record Management
- User Login
- Spending Reminder Notifications
- Export Monthly Report

## Not in MVP

The first MVP will **not** include user login, AI spending recommendations, or advanced analytics because these features require more development time and are not essential to validate the main problem. The team's focus is to create a simple and functional expense tracking system that allows users to record expenses, view spending history, and monitor their budget through a dashboard.
