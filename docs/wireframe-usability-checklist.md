# Wireframe Usability Review Checklist

Use this checklist before submitting Lab 05.

| Check Item | Yes/No | Evidence or Screenshot |
|---|---|---|
| The homepage clearly states the problem and main action. | Yes | Homepage displays the website title “Budget Tracker”, current balance and “Add Transaction” button.|
| The main user pathway can be followed from start to finish. | Yes | User flow: Homepage → Add Transaction → Transaction List → Transaction Details → Dashboard.|
| The input form includes required fields and validation feedback. | Yes | The form requires Amount , Category, Date, and Transaction Type. Validation message appears if any required field is missing.|
| The record list includes search/filter/category controls. | Yes | Transaction list includes Search, Category Filter, and Date Filter.|
| The detail view shows full information and current status. | Yes | Detail page displays Amount, Category, Date, Note, Transaction Type, and Payment Status.|
| The admin view includes at least one management action. | Yes | Admin page allows Edit/Delete Transactions and Manage Categories.|
| The dashboard shows meaningful summary information. | Yes | Dashboard displays Total Income (฿20000), Total Expenses (฿18500), Remaining Balance (฿1500) and expense charts.|
| Navigation is consistent across screens. | Yes | Navigation bar is available on every page (Home, Transactions, Dashboard, Admin).|
| The design uses realistic sample data. | Yes | Sample data uses Thai Baht, e.g. Salary +฿20000, Food -฿120, Transport -฿45, Shopping -฿800.|
| Each screen maps to system-requirements.md. | Yes | Every webpage corresponds to the functional requirements in system-requirements.md.|
