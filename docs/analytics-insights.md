# Lab 08 - Analytics Insights

## 1. Analytics Question

What did users do, say, and prove when testing the Budget Tracker MVP?

---

## 2. Metrics Calculated

| Metric | Formula / Method | Result |
|--------|------------------|--------|
| Total test users | Count testers (T001–T020) | 20 Testers |
| Completed tasks | Count TaskCompleted = Yes | 17 / 20 (85.00%) |
| Task success rate | Completed tasks / total testers | 85.00% |
| Average ease of use score | Average EaseOfUseScore (1–5) | 4.30 / 5.0 |
| Average usefulness score | Average UsefulnessScore (1–5) | 4.45 / 5.0 |
| Average interest level | Average InterestLevel (1–5) | 4.35 / 5.0 |
| Confusion points | Count repeated confusion categories | None (11), Edit button (1), Edit function (1), Search filter (1), Category filter (1), Delete confirmation (1), Search Records (1), Category Filter (1), Record List (1), Dashboard (1) |

---

## 3. Findings

1. Most users successfully completed the assigned tasks. The Budget Tracker achieved an **85.00% task success rate**, with **17 out of 20** testers completing their tasks successfully.

2. The **Expense Tracking, Income Tracking, Dashboard, Record List, and Reports** features received high usefulness scores (average **4.45/5.0**). Most users said these features were easy to understand and useful for managing daily finances.

3. A few users experienced difficulties with the **Edit Expense**, **Search Transaction**, and **Category Filter** functions. These features should be improved before the next prototype.

---

## 4. Interpretation

The testing results show that the Budget Tracker MVP successfully supports users in recording income and expenses. Most testers completed their tasks and expressed satisfaction with the system. However, improvements are still needed for editing transactions, searching records, and category filtering to increase usability.
