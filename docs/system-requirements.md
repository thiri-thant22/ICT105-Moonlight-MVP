# System Requirements

## Project Title
Budget Tracker

## Functional Requirements

| Req ID | Requirement | Description |
|---|---|---|
| FR-01 | Homepage | The system shall display a homepage with the project title, target users, and a "Get Started" button. |
| FR-02 | User Navigation | The system shall allow users to navigate from the homepage to the dashboard and expense management pages. |
| FR-03 | Add Expense | Users shall be able to enter a new expense by providing the amount, category, date, and description. |
| FR-04 | Store Expense Records | The system shall save expense records in a database, Google Sheet, or local storage. |
| FR-05 | View Expenses | Users shall be able to view all recorded expenses in a list or table. |
| FR-06 | Search and Filter | Users shall be able to search or filter expenses by category or date. |
| FR-07 | Expense Details | Users shall be able to view detailed information for each expense record. |
| FR-08 | Budget Status | The system shall display the user's current budget status, such as Remaining Budget or Budget Exceeded. |
| FR-09 | Edit/Delete Expense | Users shall be able to edit or delete incorrect expense records. |
| FR-10 | Input Validation | The system shall require mandatory fields and display an error message if information is incomplete. |
| FR-11 | Confirmation Message | The system shall display a confirmation message after an expense is successfully added, updated, or deleted. |
| FR-12 | Dashboard | The system shall display a dashboard showing total expenses, spending by category, and remaining budget. |
| FR-13 | Consistent User Interface | All pages shall use a consistent layout, colors, buttons, and navigation. |
| FR-14 | Responsive Design | The interface shall work properly on desktop and mobile devices. |
| FR-15 | Privacy | The system shall collect only necessary information and protect user data. |
| FR-16 | Requirement Traceability | Every feature in the prototype shall support the user stories and MVP feature list created in Lab 04. |

---

## Non-Functional Requirements

| Requirement | Description |
|---|---|
| Usability | The application should be simple and easy to use. |
| Performance | Pages should load quickly and respond smoothly. |
| Reliability | Expense records should be saved correctly. |
| Security | User information should be protected and not shared with others. |
| Compatibility | The system should work on modern web browsers. |
| Maintainability | The application should be easy to update with new features in the future. |

---

## Minimum Technical Requirement

### Platform
- Web Application

### Technologies
- HTML
- CSS
- JavaScript
- Local Storage or Google Sheets (for prototype)
- Figma (UI Design)

### Prototype Scope
The prototype will allow users to:
- Add a new expense
- View expense records
- Search or filter expenses
- Edit or delete expenses
- View a dashboard showing total expenses and spending categories
