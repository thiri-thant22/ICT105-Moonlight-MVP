# Critical Assumptions

## Instruction
Identify assumptions that could cause your final prototype to fail if they are wrong.

| Assumption ID | Category | Assumption | Related Requirement/User Story | Risk Level | Current Evidence | How to Test |
|---|---|---|---|---|---|---|
| A-01 | User problem | Students need an easy way to record daily income and expenses. | FR-01 / US-01 | High | Lab 03 customer interviews | User testing with target users |
| A-02 | Value proposition | Users will regularly use the Budget Tracker if it is simple and fast. | FR-02 / US-02 | High | Customer interview feedback | Prototype usability testing |
| A-03 | Usability | Users can easily understand the Dashboard summary and budget information. | FR-05 / US-04 | Medium | Wireframe review (Lab 05) | Observe users completing common tasks |
| A-04 | Technical feasibility | Browser Local Storage is sufficient for storing expense records in the MVP. | FR-06 | Medium | Prototype design | Functional testing with sample data |
| A-05 | Business logic | Expense categories (Food, Transport, Shopping, Education) are enough for most users. | FR-03 | Low | Interview results | Collect user feedback after testing |

## Categories
Use these categories:
- User problem
- Value proposition
- Usability
- Technical feasibility
- Business logic
- Data handling
