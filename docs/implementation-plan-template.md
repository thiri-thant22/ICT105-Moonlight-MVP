## 3. Implementation Approach

Explain how your prototype will be built.

- Frontend:
  HTML, CSS, and JavaScript are used to develop the Budget Tracker Website interface.

- Data source/storage:
  Transaction data is stored in local JSON files and CSV files for prototype testing.

- Admin/status handling:
  User records can be edited and deleted through the transaction management page.

- Search/filter approach:
  Users can search transactions by keyword and filter records by category and date.

- Validation approach:
  Required fields are validated before submission. Numeric fields only accept valid numbers.

- Screenshots/evidence approach:
  Progress will be documented using GitHub commits, screenshots, and weekly logbooks.
