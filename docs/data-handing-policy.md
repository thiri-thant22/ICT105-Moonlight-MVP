# Data Handling Policy

## Data Collection
The Budget Tracker prototype collects basic financial information entered by users, including expense amount, income amount, transaction category, date, and optional notes. During user validation, it also collects usability feedback such as task completion, ease of use, usefulness score, interest level, and comments.

## Data Storage
The prototype stores data in CSV files for testing and analysis. Dashboard analytics are generated from the CSV dataset using Power BI. No real database is used in the prototype stage.

## Data Access
Only the project team members can view, edit, and analyze the collected data. Test participants can only access and manage their own sample records during the validation session.

## Data Minimization
The prototype does not collect unnecessary personal information such as full name, phone number, email address, national ID number, bank account details, passwords, or precise location. Only the information required for expense tracking and usability testing is collected.

## Responsible Data Rule
The prototype avoids collecting sensitive personal data. All testing data uses sample or anonymized information, and the collected data is used only for educational purposes and prototype evaluation.
