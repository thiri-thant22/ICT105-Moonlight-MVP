# MVP Experiment Plan

## 1. Group and Project Information
- Group name: Moonlight
- Project title: Budget Tracker
- Repository link:  https://github.com/thiri-thant22
- Main target user: University students and young professionals
- Prototype platform: HTML , CSS , JavaScript (Browser Local Storage)

## 2. Experiment Objective
We want to  treat whether users can easily record expenses, view their spending summary and navigate the Budget Tracker without assistance.

---

## 3. Requirement Scope for the Experiment

| Requirement ID | Requirement Summary | Related Screen/Feature | Tested in This Experiment? |
|----------------|---------------------|------------------------|----------------------------|
| FR-01 | Dashboard | Dashboard | Yes |
| FR-02 | Add Expense | Add Expense Form | Yes |
| FR-03 | View Expense List | Expense List | Yes |
| FR-04 | Search and Filter | Search Function | Yes |
| FR-05 | Edit Expense | Edit Expense | Yes |
| FR-06 | Delete Expense | Delete Expense | Yes |

---

## 4. MVP Experiment Type

Selected experiment type:

- Clickable prototype
- Simple web prototype
- Dashboard demo

Reason for selection:

These experiment types allow users to interact with the Budget Tracker prototype, complete common budgeting tasks, and provide feedback on usability, navigation, and overall user experience before the final implementation.

---

## 5. Test Users

| Test User Group | Number of Testers | Why They Are Relevant |
|-----------------|-------------------|-----------------------|
| Students | 5 | They are the main target users of the Budget Tracker system. |

---

## 6. Experiment Procedure Summary

The testers will be asked to complete several tasks using the Budget Tracker prototype. They will add an expense, view the dashboard summary, search for a transaction, edit a record, and delete a record. During the test, the team will observe user behavior, record any problems, and collect feedback using the feedback form.

---

## 7. Expected Learning

The team expects to learn whether users can complete the main tasks without assistance, understand the dashboard information, and navigate the system easily. The feedback will be used to improve the usability and functionality of the final prototype.
